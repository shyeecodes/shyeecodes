## Repository structure
Each top-level folder represents a project or category. Example layout:
- `projects/` — focused apps and experiments
- `libs/` — reusable libraries or utilities
- `docs/` — documentation, notes, and resources
- `website/` — personal site or demo pages
- `README.md` — this file

Note: See each project's README for exact language, dependencies, and run commands.

## Getting started
Prerequisites
- Git: to clone the repository
- Language runtimes and tools: node, python, or other runtimes depending on the project (see project README)

Clone the repo
```bash
git clone https://github.com/shyeecodes/shyeecodes.git
cd shyeecodes
```

Install and run a project
1. Change into the project's folder:
```bash
cd projects/<project-name>
```
2. Follow that project's README for installation and run commands (for example `npm install && npm start` or `pip install -r requirements.txt`).

Testing
- Follow the testing instructions per project. Most projects include a `test` script in their README.

## Contributing
Contributions are welcome! To contribute:
1. Fork the repository.
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Commit your changes and push: `git push origin feature/your-feature`
4. Open a Pull Request describing your change.

Please open an issue first if your change is large or you want to propose a new project.

Code style and guidelines
- Keep commits small and focused.
- Add or update tests for new functionality where appropriate.
- Document setup steps in the project's README.
